GitHub Pages 404 Error

Solving the "File not found" error when uploading your resume

Understanding the Problem

You're seeing a 404 error on GitHub Pages because the server can't find the file you're trying to access. This typically happens due to one of these reasons:

Common Causes:

  • Incorrect file name or case sensitivity issues
  • Missing index.html file for root URL
  • File permissions preventing access
  • Incorrect repository configuration

Step-by-Step Solution

1. Check File Name and Case Sensitivity

GitHub Pages is case-sensitive. Make sure your file name matches exactly with the URL, including capitalization.

// If your resume is named "MyResume.pdf"
// But you're linking to "myresume.pdf"
// This will cause a 404 error

2. Ensure Proper File Structure

Your repository should have the correct structure. For a resume, you might have:

your-username.github.io/
├── index.html # Your main page
├── resume.pdf # Your resume file
└── assets/
├── css/style.css
└── images/logo.png

3. Enable GitHub Pages for Your Repository

Make sure GitHub Pages is enabled in your repository settings:

  1. Go to your repository on GitHub
  2. Click on "Settings"
  3. Scroll down to the "GitHub Pages" section
  4. Select the source branch (typically main or gh-pages)
  5. Click "Save"

4. Create an index.html File (If Missing)

For root URLs, you need an index.html file. Create one with this basic template:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Resume</title>
</head>
<body>
    <h1>Welcome to My Portfolio</h1>
    <p>View my <a href="resume.pdf">resume</a></p>
</body>
</html>

Important Note

After making changes to your repository, it may take a few minutes for GitHub Pages to update. Be patient and refresh your page after 2-3 minutes.

Testing Your Solution

Once you've implemented the changes:

  1. Commit and push your changes to GitHub
  2. Wait a few minutes for GitHub Pages to rebuild your site
  3. Refresh your GitHub Pages URL
  4. If the error persists, double-check the file names and repository settings
